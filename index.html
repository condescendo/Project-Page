Core Objective

This project engineers a low-level extension to the Global Illumination Gaussian Splatting (GI-GS) pipeline. By mathematically decomposing scenes into discrete "Object" and "Background" entities at the rasterization level, this system enables high-fidelity asset extraction and physically-based relighting.

Asset Extraction

Clean separation of geometry without background artifacts using atomic visibility checks.

Scene Compositing

Placing extracted objects into novel environments with accurate occlusion handling.

PBR Relighting

Solves for Albedo, Roughness, and Metallic properties, allowing objects to react realistically to new lighting.

Systems Engineering & CUDA Architecture

The primary contribution is a modification of the underlying C++/CUDA rasterization backend (based on diff-gaussian-rasterization) to support dynamic visibility tracking and memory management.

1. Custom CUDA Kernels

To achieve object isolation without expensive post-processing, I rewrote the core renderCUDA kernel. Instead of standard alpha blending, the modified forward pass performs atomic checks directly on the GPU.

  • Device Memory Management: Allocated a custom boolean tensor (seen) on the GPU to track visibility persistence.
  • Zero-Copy Logic: Visibility logic remains entirely on the VRAM during the forward pass, syncing to the host only for topological changes.

Optimization & Data Flow

Occlusion-Aware Pruning

Implemented a Python-side controller that aggregates the CUDA-generated seen tensor via C++ bindings. Points that fail to register a true status in the GPU buffer are pruned, reducing VRAM usage and eliminating internal geometry artifacts.

Masked PBR Loss

Engineered a custom loss pipeline that applies segmentation masks before gradient calculation. This stabilizes the non-convex optimization of PBR parameters (Albedo, Roughness) by preventing the solver from fitting noise in empty space.
