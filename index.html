Object-Centric
Inverse Rendering

A Hybrid Python/CUDA Pipeline

Context & Motivation

3D Gaussian Splatting (3DGS) has revolutionized view synthesis with real-time inference speeds. However, the standard implementation relies on "baked radiance," meaning lighting and geometry are fused together.

Existing Capability

Standard 3DGS offers high-fidelity reconstruction, but treats the scene as a static photo. Objects cannot be moved or relit without breaking realism.

Detected Limitation (Target)

Current Inverse Rendering methods solve the scene globally. They fail to distinguish the Object of Interest from background noise, resulting in "floaters" and dirty albedo maps.

Figure 1: The challenge of separating object from environment

Solution Architecture

This project bridges the gap by injecting Object-Centric Masking into the PBR optimization loop. It ensures that material properties (Albedo, Roughness) are only solved for valid geometry.

1. Logic Flow (Python)

Re-architected the train.py loop to enforce segmentation masks during the PBR stage. This prevents the optimizer from "hiding" loss in the background pixels.

train.py (Snippet) 
# PBR Stage: Optimization of Material Properties
if dataset.use_object_mask:
    object_mask = viewpoint_cam.object_mask.cuda()
    
    # Mask the Ground Truth and the Prediction
    # This ensures gradients are only calculated for the object
    gt_image = gt_image * object_mask
    pred_albedo = pred_albedo * object_mask

pbr_loss = l1_loss(pred_albedo, gt_image)

Key Results

/ Result 01
Occlusion-Aware Pruning

Algorithms automatically detect and delete internal geometry hidden inside the object, resulting in hollow, clean meshes.

/ Efficiency
~40%

Reduction in VRAM usage during the PBR training stage due to aggressive culling of unseen points.

/ Result 02
Artifact-Free Relighting

Extracted albedo maps are devoid of baked shadows, allowing for convincing compositing into HDR environments.

/ Pipeline
Pybind11 Integration

Seamless bridge allowing Python to control high-level topology logic while C++ handles the heavy pixel-wise atomic operations.
